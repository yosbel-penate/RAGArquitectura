Información:

Descripción: Introduce el concepto de flujo (stream) como la forma general de manejar operaciones de entrada y salida, donde la información se maneja como secuencias de bytes.
6. Diapositiva: Flujo (Entrada) (Página 6)

Información:
Definición: En las operaciones de entrada, los bytes fluyen de un dispositivo (como un teclado, una unidad de disco, una conexión de red) a la memoria principal.
Descripción: Describe el flujo en las operaciones de entrada, que ocurre cuando los bytes se transfieren desde un dispositivo externo a la memoria principal.
7. Diapositiva: Flujo (Salida) (Página 7)

Información:
Definición: En las operaciones de salida, los bytes fluyen de la memoria principal a un dispositivo (como una pantalla, una impresora, una unidad de disco, una conexión de red).
Descripción: Describe el flujo en las operaciones de salida, que ocurre cuando los bytes se transfieren desde la memoria principal a un dispositivo externo.

13. Diapositiva: Captura de Datos (InputStream.read() - Conversión) (Página 13)

Información:
Definición: Este método lee un carácter por cada Ilamada y retorna un int. Si se espera otro tipo hay que hacer una conversión explícita mediante un cast.
Ejemplo: char c = (char)System.in.read();
Descripción: Explica cómo el método read() lee un carácter y retorna un int, y cómo debe realizarse un cast para convertir el resultado a un char.
14. Diapositiva: Captura de Datos (Lectura de Cadenas - Bucle) (Página 14)

Información:
Definición: Para leer datos más largos que un simple carácter es necesario emplear un bucle while o for y unir los caracteres.
Ejemplo de código:
```java
    char c;
    String frase = new String("");
    while((c=System.in.read()) != '\n')
        frase = frase + c; // frase.append(c);
```
Descripción: Muestra la necesidad de usar un bucle para leer cadenas de caracteres, uniendo los caracteres leídos.
