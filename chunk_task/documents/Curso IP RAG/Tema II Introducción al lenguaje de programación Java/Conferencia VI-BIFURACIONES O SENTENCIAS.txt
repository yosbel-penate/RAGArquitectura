Información:

Descripción: Explica la ejecución secuencial, que es el comportamiento por defecto de los programas, donde las instrucciones se ejecutan en orden.

Descripción: Introduce el concepto de transferencia de control, donde el orden de ejecución de las instrucciones puede variar.

12. Diapositiva: Bifurcación if (Introducción) (Página 12)

Información:
Definición: Permite ejecutar un conjunto de sentencias en función del valor que tenga la expresión de comparación (se ejecuta si es true).
Descripción: Introduce el bucle if, explicando que permite ejecutar un bloque de código solo si la expresión booleana asociada se evalúa como verdadera.
13. Diapositiva: Bifurcación if (Sintaxis) (Página 13)

Información:
Sintaxis:
    if (booleanExpression) {
       statements;
    }
Texto: Las llaves {} sirven para agrupar en un bloque las sentencias que se han de ejecutar, y no son necesarias si sólo hay una sentencia dentro del if.
Descripción: Muestra la sintaxis del if, explicando que las llaves se utilizan para agrupar múltiples sentencias y son opcionales si hay una sola instrucción.

15. Diapositiva: Bifurcación if else (Introducción) (Página 15)

Información:
Definición: Análoga a la anterior, de la cual es una ampliación. Las sentencias incluidas en el else se ejecutan en el caso de no cumplirse la expresión de comparación (false).
Descripción: Introduce la bifurcación if else, una extensión del if donde se especifica un bloque de código alternativo a ejecutar cuando la condición es falsa.
16. Diapositiva: Bifurcación if else (Sintaxis) (Página 16)

Información:
Sintaxis:
    if (booleanExpression) {
        statements1;
     } else {
       statements2;
    }
Descripción: Muestra la sintaxis de if else, donde un bloque de código (statements1) se ejecuta si la condición es verdadera y un bloque alternativo (statements2) se ejecuta si la condición es falsa.

19. Diapositiva: Bifurcación if elseif else (Introducción) (Página 19)

Información:
Definición: Permite introducir más de una expresión de comparación. Si la primera condición no se cumple, se compara la segunda y así sucesivamente.
Descripción: Introduce la bifurcación if elseif else, explicando que permite introducir múltiples condiciones que serán evaluadas secuencialmente.
20. Diapositiva: Bifurcación if elseif else (Sintaxis) (Página 20)

Información:
Sintaxis:
    if (booleanExpression1) {
       statements1;
     } else if (booleanExpression2) {
       statements2;
     } else if (booleanExpression3) {
         statements3;
     } else {
         statements4;
    }
Descripción: Presenta la sintaxis del if elseif else, mostrando cómo se encadenan múltiples condiciones y la forma de incluir un else opcional al final.

23. Diapositiva: Sentencia switch (Introducción) (Página 23)

Información:
Definición: Es una alternativa a la bifurcación if elseif else cuando se compara la misma expresión con distintos valores.
Descripción: Introduce la sentencia switch, como una alternativa a if elseif else cuando se necesita comparar una expresión con diferentes valores concretos.
24. Diapositiva: Sentencia switch (Sintaxis) (Página 24)

Información:
Sintaxis:
switch (expression) {
      case value1: statements1; break;
      case value2: statements2; break;
      case value3: statements3; break;
      case value4: statements4; break;
      case value5: statements5; break;
      default: statements6; break;
  }
Descripción: Presenta la sintaxis general de la sentencia switch, mostrando cómo se utilizan case y default.
25. Diapositiva: Sentencia switch (Características - case) (Página 25)

Información:
Definición: Cada sentencia case se corresponde con un único valor de expresión. No se pueden establecer rangos sino que se debe comparar con valores concretos.
Descripción: Explica que en switch, cada case corresponde a un valor específico y que no se pueden comparar rangos.
26. Diapositiva: Sentencia switch (Características - default) (Página 26)

Información:
Definición: Los valores no comprendidos en ninguna sentencia case se pueden gestionar en default, que es opcional.
Descripción: Explica la funcionalidad de default, que es un case opcional que se ejecuta si ningún valor coincide en los case.
27. Diapositiva: Sentencia switch (Características - break) (Página 27)

Información:
Definición: En ausencia de break, cuando se ejecuta una sentencia case se ejecutan también todas las case que van a continuación, hasta que se llega a un break o hasta que se termina el switch.
Descripción: Indica el comportamiento de switch cuando no se incluye la sentencia break, donde se ejecutan todos los case a partir del que coincide.
28. Diapositiva: Sentencia switch (Ejemplo - Explicación) (Página 28)

Información:
Ejemplo con explicación:
char c = (char)(Math.random()%26+'a'); /* Generar
   una letra minusculas*/
System.out.println("La letra " + c);
switch (c) {
   case 'a': // Se compara con la letra a
   case 'e': // Se compara con la letra e
   case 'i': // Se compara con la letra i
   case 'o': // Se compara con la letra o
   case 'u': // Se compara con la letra u
      System.out.println(" Es una vocal "); break;
   default:
       System.out.println(" Es una consonante ");
  }
Descripción: Muestra un ejemplo donde se utiliza la sentencia switch para determinar si una letra generada aleatoriamente es una vocal o no, mostrando las consecuencias de la falta de break.

33. Diapositiva: Operador condicional(?:) (Introducción) (Página 33)

Información:
Definición: Java cuenta con el operador condicional (?:), que en ocasiones puede utilizarse en lugar de una instrucción if...else. Éste es el único operador ternario en Java.
Descripción: Introduce el operador condicional (?:), que es un operador ternario, o sea, que opera con tres operandos.
34. Diapositiva: Operador condicional(?:) (Estructura 1) (Página 34)

Información:
Definición: En conjunto, los operandos y el símbolo ?: forman una expresión condicional. El primer operando (a la izquierda del ? ) es una expresión booleana.
Descripción: Explica que el operador condicional (?:) requiere tres operandos y que el primero de ellos debe ser una expresión booleana.
35. Diapositiva: Operador condicional(?:) (Estructura 2) (Página 35)

Información:
Definición: El segundo operando (entre el ? y :) es el valor de la expresión condicional si la expresión booleana es verdadera.
Descripción: Explica que el segundo operando del operador condicional (?:) es el valor que retornará si la condición es true.
36. Diapositiva: Operador condicional(?:) (Estructura 3 y Ejemplo) (Página 36)

Información:
Definición: El tercer operando (a la derecha de :) es el valor de la expresión condicional si la expresión booleana se evalúa como false.
Ejemplo:
    System.out.println(calificacion >=3 ? "Aprobado":"Reprobado");
Descripción: Explica que el tercer operando del operador condicional (?:) es el valor que retornará si la condición es false y presenta un ejemplo.
